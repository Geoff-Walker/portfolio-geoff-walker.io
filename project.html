The Problem

Aspects of Beauty is a small family run beauticians in Lincolnshire, who's owner came to me because their webpage wouldn't open properly in mobile and was nearly unsuable. It had been made by a family friend of the owner and was a poor example of a no code website!

The Solution

I took the existing feel of the design and came up with an exhaustive list of things that needed changing from the original. I then rebuilt the site from the ground up fixing the errors as I went. These included simple things like not having more than two fonts on a page or making sure the right images or logos are shown. Through setting up a CSS-Grid with some repeat minmax so all screen sizes would be accomodated. The biggest change I made to the site was the landing section which previosuly had been really dull but I added some animated images behind a Mask-Image as shown in the preview.

The Problem

A fitness tracker that displays hours doing different acitivities using data coming from a Local JSON file.

The Solution

The design of this project is based upon the Front End Mentor challenge Time Tracking Dashboard. I love Front End Mentor as it provides the design and the problem but leaves it entirely up to you how to achieve this much more inline with corporate work, while providing a platform to test your skills.

I used CSS-Grid and repeat minmax to create a fully responsive activity tracker with media queries for phone, tablet and desktop. I made this tracker exclusively in the mobile first workflow

The Problem

To build an etch a sketch appliation in HTML, CSS and Vanilla Javascript

The Solution

The etch a sketch application was build using CSS-Grid to create a grid of divs based upon the number specified using css variables.

I also used three random number generators to create a red, green and blue variables which combined gives a random rainbow affect.

The very simple mousover event then applies the colour to the sketch pad

The Problem

Building projects with an APi can be a minefield not so much the build process but successful free API's soon become not free when enough people use them. I need an Api that I can rely on being around for at least a couple of years.

The Solution

To that end I have used the API created by Jonas Schmedtmann for use in his Master Javascript Udemy course to build an application using the javascript module-pattern to render the results. The APi allows me to both read hundreds of recipes from it but also write my personal recipes to it via an ssh key.
